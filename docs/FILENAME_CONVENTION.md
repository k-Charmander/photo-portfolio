# 파일명 규칙 (Google Drive)

사진 소스는 **Google Drive 폴더**입니다. EXIF가 없을 수 있으므로 지역·테마·설명의 1차
정보원은 **파일명**입니다. Drive 폴더에 올리는 사진 파일명을 아래 형식으로 지어 주세요.

## 형식

```
<지역>_<테마>_<자유설명>.jpg
```

밑줄(`_`)로 구분합니다. 예시:

```
제주_풍경_성산일출봉 위로 번지던 아침 바다.jpg
서울_야경_한강 위로 흐르던 도시의 불빛.jpg
부산_음식_골목 노포의 따뜻한 한 끼.jpg
```

## 규칙

- **지역**: 인식 값 — `제주 · 서울 · 인천 · 부산 · 강원 · 경주` (영문 별칭 `jeju/seoul/busan…`도 가능).
  사진에 GPS가 살아 있으면 그 좌표가 우선합니다. 못 찾으면 `미상`.
- **테마**: 인식 값 — `풍경 · 노을 · 야경 · 거리 · 인물 · 음식 · 일상`.
  지정이 없으면 자유설명·시간대로 추정하고, 그래도 없으면 `일상`.
- **자유설명**: 지역·테마 토큰을 뺀 나머지가 스토리 본문에 쓰입니다. 공백 포함 가능.
  비워 두면 테마별 기본 문장이 들어갑니다.
- 토큰 순서는 자유롭지만 `지역_테마_설명` 순서를 권장합니다. 지역/테마는 표와 **정확히 일치**할 때만
  인식되고, 일치하지 않는 토큰은 자유설명으로 처리됩니다.

> 해시태그는 베이스 태그 + 지역/테마 태그가 자동으로 붙습니다(파일명에 `#`를 적을 필요 없음).

## 동작 방식

매일 오전 10시(GitHub Actions) 파이프라인이 Drive 폴더에서 신규 사진을 가져와:

1. 파일명을 파싱(지역/테마/설명)하고 EXIF/GPS가 있으면 보강
2. 지역·테마로 분류
3. 템플릿으로 한국어 스토리·해시태그 생성
4. `data/photos.json` 갱신 → 사이트 아카이빙
5. 인스타 게시 초안(이미지 + 캡션)을 만들고 GitHub Issue로 알림

> 로컬 점검: `npm run pipeline:sample` (네트워크 없이 `data/sample/album.json`으로 실행)
