# 다음 세션 TODO — 디자인을 핀터레스트 스타일로 재작업

> 상태: **요청됨** (다음 세션에서 진행). 사용자가 "전체 디자인을 핀터레스트 스타일로,
> 폰트·레이아웃 등 손볼 것"이라고 요청함.

## 현재 디자인 (출발점)

- Figma 라이트/파스텔 **컬러블록** 시스템 — `docs/DESIGN.md`.
- 화이트 캔버스 + 검정 잉크, 지역별 파스텔 컬러블록(lime/lilac/mint/cream…),
  pill 버튼, **mono eyebrow**(JetBrains Mono 대문자 라벨), 본문 Pretendard.
- 아카이브는 이미 메이슨리(컬럼) 그리드.

## 목표 — 핀터레스트 스타일

전체를 핀터레스트(Pinterest) 느낌으로 재작업. 핵심 특징:

### 레이아웃

- **메이슨리 그리드를 메인(첫 화면)으로** — 이미지 우선, 촘촘한 다열 카드.
- 둥근 모서리 카드, **가벼운 화이트/연회색 배경**, 미니멀한 헤더(검색 느낌의 얇은 상단바).
- **파스텔 컬러블록(지역 패널) 축소/제거** — 핀터레스트는 색 패널이 아니라 사진 자체가 주인공.
- 카드 호버 시 **저장/캡션 오버레이**(핀터레스트의 Save 버튼 느낌), 부드러운 그림자.
- 지역/테마는 상단 **필터 칩/탭** 정도로 가볍게.

### 폰트

- **단일 산세리프 보이스**로 정리(핀터레스트는 깔끔한 sans). 본문 Pretendard 유지 가능.
- **mono eyebrow 라벨 제거/완화** — 핀터레스트엔 mono 대문자 라벨이 어울리지 않음.
- 큰 디스플레이 헤드라인보다 **카드 중심의 작은 타이포** 위주.

### 카드/디테일

- 사진 중심, 캡션·지역·테마는 작게(카드 하단 또는 호버).
- 라이트박스(상세)는 유지하되 핀터레스트 핀 상세처럼(좌측 큰 이미지 + 우측 정보).

## 작업 대상 파일

- `index.html`, `css/style.css`, `js/main.js` (데이터 흐름·파이프라인은 변경 없음).
- 디자인 토큰 출발점: `docs/DESIGN.md`. 이 문서의 방향으로 재작업 후 `docs/DESIGN.md` 갱신 검토.

## 검증

- `npm test`(format + lint) 통과, `npm run dev` 프리뷰 + 스크린샷 확인.
- 브랜치 작업 후 PR → `main` 머지로 라이브 반영(GitHub Pages).
