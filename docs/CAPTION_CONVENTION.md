# 캡션 컨벤션 (iCloud 공유 앨범)

iCloud 공유 앨범은 사진의 **EXIF/GPS를 대부분 제거**합니다. 그래서 지역·테마 분류와
스토리의 1차 정보원은 **사진에 적는 캡션**입니다. 아이폰 공유 앨범에서 각 사진에 아래
형식으로 캡션을 적어 주세요.

## 형식

```
<지역> <장소(선택)> #<테마> #<추가태그…> <자유 설명>
```

예시:

```
제주 성산일출봉 #풍경 #일출 성산일출봉 위로 번지던 아침 바다
서울 한강 #야경 한강 위로 흐르던 도시의 불빛
부산 #음식 #foodie 골목 노포의 따뜻한 한 끼
```

## 규칙

- **지역**: 맨 앞에 한 단어. 인식되는 값 — `제주 · 서울 · 인천 · 부산 · 강원 · 경주`
  (영문 별칭 `jeju/seoul/busan…`도 가능). GPS 좌표가 살아 있으면 그 값이 우선합니다.
  지역을 못 찾으면 `미상`으로 분류됩니다.
- **테마**: `#해시태그`로 지정. 인식 테마 — `풍경 · 노을 · 야경 · 거리 · 인물 · 음식 · 일상`.
  지정이 없으면 자유 설명의 키워드와 촬영 시간대로 추정하고, 그래도 없으면 `일상`.
- **해시태그**: `#` 로 시작하는 모든 토큰은 그대로 게시 해시태그에 포함됩니다.
  여기에 베이스 태그(`#사진 #photography …`)와 지역·테마 태그가 자동으로 더해집니다.
- **자유 설명**: 해시태그와 지역 단어를 뺀 나머지 문장이 스토리 본문에 사용됩니다.
  비워 두면 테마별 기본 문장이 들어갑니다.

## 동작 방식

매일 오전 10시(GitHub Actions) 파이프라인이 공유 앨범에서 신규 사진을 가져와:

1. 캡션을 파싱하고(지역/테마/태그/설명) EXIF가 있으면 보강
2. 지역·테마로 분류
3. 템플릿으로 한국어 스토리와 해시태그 생성
4. `data/photos.json` 갱신 → 사이트에 아카이빙
5. 인스타 게시 초안(이미지 + 캡션)을 만들고 GitHub Issue로 알림

> 로컬 점검: `npm run pipeline:sample` (네트워크 없이 `data/sample/album.json`으로 실행)
